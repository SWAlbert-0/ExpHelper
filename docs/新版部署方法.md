# 实验助手新版部署方法（精简版）

本文档只保留最短可运行路径。目录规范：
- `docs/` 仅放文档
- `docker/` 放所有部署相关文件
- 后端构建运行环境：`Java 17`

## 1. 快速启动（推荐）

### 第 1 步：准备环境变量
```powershell
cd E:\Project\ExpHelper\docker
copy .env.example .env
```

### 第 2 步：构建后端 jar
```powershell
cd E:\Project\ExpHelper\exphlp
mvn -pl api/webApp -am clean package -DskipTests
```
产物必须存在：`exphlp/api/webApp/target/webApp-1.0-SNAPSHOT.jar`

### 第 3 步：执行预检（强烈推荐）
```powershell
cd E:\Project\ExpHelper\docker
powershell -ExecutionPolicy Bypass -File .\scripts\preflight.ps1
```

### 第 4 步：一键启动
```powershell
cd E:\Project\ExpHelper\docker
docker compose --env-file .env up -d --build
```

### 第 5 步：验证
```powershell
docker compose ps
curl.exe -i http://localhost:8086/api/auth/healthz
curl.exe -i http://localhost:8086/api/auth/captcha
cd E:\Project\ExpHelper\docker
powershell -ExecutionPolicy Bypass -File .\scripts\smoke.ps1 -ApiBase http://localhost:8080
```
访问：`http://localhost:8086/#/index`

默认账号：
- 用户名：`admin`
- 密码：`123456`

## 2. 网络受限兜底（仅当前端镜像构建失败时）

1. 本机构建前端
```powershell
cd E:\Project\ExpHelper\exphlp-front
npm install
npm run build:prod
```

2. 使用本地 `dist` 启动
```powershell
cd E:\Project\ExpHelper\docker
docker compose --env-file .env -f docker-compose.local-front.yml up -d --build
```

## 3. 常见故障速查

1. `webapp` 启动失败：先确认第 2 步 jar 是否存在。  
2. 页面提示“后端接口连接异常”：先看日志  
```powershell
docker logs --tail 200 c_exphlp_webapp
```
3. 前端构建出现 `ECONNREFUSED 127.0.0.1:xxxx`：不要用主机回环代理，改用兜底方案。  
4. 登录后 401：确认请求头带 `Authorization: Bearer <token>`。  
5. 点击执行计划后提示未受理：查看 `execute` 返回 `data.accepted`，常见原因是“计划执行中/队列已满”。
