# 接口总览（版本2）

## 一、当前业务主链路接口（`/api/*`）

### 认证与个人中心

- `POST /api/auth/login`
- `GET /api/auth/me`
- `GET /api/auth/profile`
- `PUT /api/auth/profile`
- `PUT /api/auth/password`
- `POST /api/auth/avatar`
- `GET /api/auth/avatar/{filename}`
- `POST /api/auth/logout`
- `GET /api/auth/healthz`

### 执行计划

- `POST /api/ExePlanController/execute`
- `GET /api/ExePlanController/getExePlans`
- `POST /api/ExePlanController/addExePlan`
- `POST /api/ExePlanController/deleteExePlanById`
- `POST /api/ExePlanController/updateExePlanById`

### 平台用户管理

- `GET /api/PlatController/getUsersByPage`
- `GET /api/PlatController/getUserByName`
- `POST /api/PlatController/addUser`
- `POST /api/PlatController/updateUserById`
- `POST /api/PlatController/deleteUserById`
- `POST /api/PlatController/resetUserPassword`

## 二、`/admin/*` 模板残留现状

- 当前代码中 `/admin/*` 仅保留 1 处请求层拦截哨兵：`src/utils/request.js`。
- 历史模板页面 `views/vadmin/{monitor,system,tool,permission/*}` 已下线，个人中心已迁移到 `views/profile`。
- 历史 `/admin/*` API 模块已分批删除，避免误调用进入失效链路。
- 残留基线与校验策略见 `docs/admin-api-legacy-inventory-v2.md` 与 `scripts/check-p0.ps1`。

## 三、最小链路 smoke 覆盖

`docker/scripts/smoke.ps1` 当前覆盖：

1. `GET /api/auth/healthz`
2. `GET /api/auth/captcha`
3. `POST /api/auth/login`
4. `GET /api/auth/me`（带 token）
5. `POST /api/auth/avatar`（带 token）
6. `POST /api/ExePlanController/execute?planId=bad-id`（带 token，断言错误码）
