# 字段与参数说明（Python MOEA/D 案例）

## 算法字段

- `algName`：算法展示名，建议与服务名接近。
- `serviceName`：Nacos 注册名，必须一致。
- `runtimeType`：必须为 `python`。

## exphlp-alg.json

- `runtimeType`：固定 `python`。
- `serviceName`：建议必填，防止上传契约校验误差。
- `port`：容器监听端口。
- `entry`：FastAPI 启动入口。

## 参数建议

- `nVars`：LSMOP 维度规模，建议从 300~800 递增。
- `populationSize`：建议 100~250。
- `maxGenerations`：建议 150~350。
- `maxReturnedPoints`：建议 80~140。
- `seed`：long，保证复现实验。

## 常见失败

- `ALG_CALL_TIMEOUT`：参数过大导致调用超时，先降低规模。
- `Nacos中无可用实例`：服务未注册或容器异常退出。
