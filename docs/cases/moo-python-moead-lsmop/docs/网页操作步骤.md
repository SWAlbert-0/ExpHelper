# 网页操作步骤（Python MOEA/D 案例）

## 1. 导入问题实例

1. 打开“问题实例管理”。
2. 点击“JSON导入”，拖拽 `problem_instances/lsmop-py-caseB.json`。
3. 确认实例名 `lsmop-py-caseB`。

## 2. 导入算法

1. 打开“算法库管理”。
2. 点击“JSON导入”，拖拽 `algorithm_instances/py-moead-lsmop-ls-default.json`。
3. 确认算法行：
   - 算法名称：`py-moead-lsmop-ls`
   - 服务名：`py-moead-lsmop-ls`
   - 运行时：`Python`

## 3. 上传源码并构建

1. 在算法行点击“源码”。
2. 执行：
   `powershell -ExecutionPolicy Bypass -File docs/cases/moo-python-moead-lsmop/scripts/package-source-zip.ps1`
3. 上传 `py-moead-lsmop-ls-source.zip`。
4. 点击“上传源码 -> 构建并启动”，等待成功。

## 4. 执行计划

1. 在执行向导中选择 `lsmop-py-caseB`。
2. 选择算法 `py-moead-lsmop-ls`。
3. 先执行检查，再提交执行。

## 5. 验收

- 执行日志出现 `ALG_CALL` 与 `PLAN_DONE`。
- 结果可见 `Spread/Spacing`；与其他算法放在同一计划时可观察 Coverage 对比。
