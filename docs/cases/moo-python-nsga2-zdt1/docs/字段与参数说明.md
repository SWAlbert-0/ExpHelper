# 字段与参数说明（Python NSGA-II 案例）

## 算法字段

- `algName`：算法展示名称，建议与服务名一致便于排查。
- `serviceName`：Nacos 注册名，必须与算法服务实际注册名一致。
- `runtimeType`：必须为 `python`，否则上传构建会被判定不一致。

## exphlp-alg.json

- `runtimeType`：运行时类型，固定 `python`。
- `serviceName`：用于契约校验，建议必填。
- `port`：算法容器监听端口。
- `entry`：FastAPI 启动入口，默认 `main:app`。

## 参数建议

- `nVars`：问题规模，越大越慢。
- `populationSize`：种群规模，建议 80~300。
- `maxGenerations`：迭代代数，建议先用 100~300 验证链路。
- `maxReturnedPoints`：返回前沿点数量，建议 60~120。
- `seed`：随机种子（long），用于可复现。

## 常见失败

- `runtimeType不一致`：算法库与 zip 元数据不一致。
- `serviceName不一致`：算法库与 zip 元数据不一致。
- `Nacos中无可用实例`：容器未启动或服务未注册。
