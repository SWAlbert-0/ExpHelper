# 真实案例测试文档（Java 算法 + 问题入库 + 计划执行）

本文档给出一套可复现的端到端测试：
1. 启动实验助手平台
2. 启动一个 Java 算法服务（注册到 Nacos）
3. 用 API 完成问题入库、算法入库、计划创建与执行
4. 验证执行结果

## 1. 前置条件

- 平台已按 `docs/新版部署方法.md` 启动成功
- 可访问：
  - `http://localhost:8086/#/index`
  - `http://localhost:8080/api/auth/healthz`
- 本机安装 JDK 17、Maven

### 2.3 `src/main/java/demo/AlgApp.java`
```java
package demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@SpringBootApplication
@RestController
public class AlgApp {
    public static void main(String[] args) {
        SpringApplication.run(AlgApp.class, args);
    }

    @PostMapping("/myAlg/")
    public List<EachResult> run(@RequestBody AlgRunCtx ctx) {
        List<EachResult> out = new ArrayList<>();
        String probName = ctx.probInst == null ? "unknown" : ctx.probInst.instName;
        out.add(new EachResult("case", probName, "string"));
        out.add(new EachResult("score", "100", "int"));
        return out;
    }

    public static class AlgRunCtx {
        public ProbInst probInst;
        public List<RunPara> runParas;
    }

    public static class ProbInst {
        public String instId;
        public String instName;
    }

    public static class RunPara {
        public int paraId;
        public String paraName;
        public String paraType;
        public String paraValue;
    }

    public static class EachResult {
        public String key;
        public String value;
        public String dataType;

        public EachResult() {}
        public EachResult(String key, String value, String dataType) {
            this.key = key;
            this.value = value;
            this.dataType = dataType;
        }
    }
}
```

## 3. 执行真实案例（API）

在 PowerShell 执行以下脚本：

```powershell
# 1) 登录
$loginBody = @{ username='admin'; password='123456' } | ConvertTo-Json
$login = Invoke-RestMethod -Method Post -Uri 'http://localhost:8080/api/auth/login' -ContentType 'application/json' -Body $loginBody
$token = $login.data.token
$headers = @{ Authorization = "Bearer $token" }

# 2) 入库问题实例（使用时间戳避免重名）
$suffix = Get-Date -Format 'yyyyMMddHHmmss'
$probName = "demo-prob-$suffix"
$probBody = @{
  categoryName = 'demo-cat'
  instName = $probName
  machineIp = '127.0.0.1'
  dirName = 'demo/input'
  machineName = 'local'
  description = 'demo problem'
} | ConvertTo-Json
Invoke-RestMethod -Method Post -Uri 'http://localhost:8080/api/ProbController/addProblem' -Headers $headers -ContentType 'application/json' -Body $probBody | Out-Null
$probList = Invoke-RestMethod -Method Get -Uri "http://localhost:8080/api/ProbController/getProblemsByName?probName=$probName&pageNum=1&pageSize=10" -Headers $headers
$prob = $probList | Where-Object { $_.instName -eq $probName } | Select-Object -First 1
if (-not $prob) { throw "problem not found by exact name: $probName" }
$probId = $prob.instId
$probCheck = Invoke-RestMethod -Method Get -Uri "http://localhost:8080/api/ProbController/getProblemById?proId=$probId" -Headers $headers
if (-not $probCheck -or -not $probCheck.instId) { throw "problem not found by id: $probId" }

# 3) 入库算法（serviceName 必须等于 Java 服务名 demo-java-alg）
$algName = "demo-java-alg-$suffix"
$algBody = @{
  algName = $algName
  serviceName = 'demo-java-alg'
  description = 'demo java algorithm service'
  defParas = @(
    @{ paraId = 1; paraName = 'alpha'; paraType = 'double'; paraValue = '0.5'; description = 'demo para' }
  )
} | ConvertTo-Json -Depth 6
Invoke-RestMethod -Method Post -Uri 'http://localhost:8080/api/AlgController/addAlg' -Headers $headers -ContentType 'application/json' -Body $algBody | Out-Null
$algList = Invoke-RestMethod -Method Get -Uri "http://localhost:8080/api/AlgController/getAlgsByName?algName=$algName&pageNum=1&pageSize=10" -Headers $headers
$alg = $algList | Where-Object { $_.algName -eq $algName } | Select-Object -First 1
if (-not $alg) { throw "algorithm not found by exact name: $algName" }

# 4) 创建执行计划
$planName = "demo-plan-$suffix"
$planBody = @{
  planName = $planName
  probInstIds = @($probId)
  algRunInfos = @(
    @{
      algRunInfoId = '1'
      algId = $alg.algId
      algName = $alg.algName
      serviceName = $alg.serviceName
      runNum = 1
      runParas = @(
        @{ paraId = 1; paraName = 'alpha'; paraType = 'double'; paraValue = '0.5'; description = 'demo para' }
      )
    }
  )
  userIds = @()
  exeStartTime = 0
  exeEndTime = 0
  exeState = 1
  description = 'demo e2e plan'
} | ConvertTo-Json -Depth 8
# 某些情况下 addExePlan 会因重名或服务端异常返回 500，这里增加兜底读取
$planId = $null
try {
  $planId = Invoke-RestMethod -Method Post -Uri 'http://localhost:8080/api/ExePlanController/addExePlan' -Headers $headers -ContentType 'application/json' -Body $planBody
} catch {
  Write-Host "addExePlan failed, fallback to getExePlanByName..."
}
if (-not $planId) {
  $plan = Invoke-RestMethod -Method Get -Uri "http://localhost:8080/api/ExePlanController/getExePlanByName?planName=$planName" -Headers $headers
  $planId = $plan.planId
}
if (-not $planId) {
  throw "planId is empty, stop here and check backend logs: docker logs --tail 300 c_exphlp_webapp"
}
Write-Host "planId=$planId"

# 5) 执行计划
$exec = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/ExePlanController/execute?planId=$planId" -Headers $headers
$exec | ConvertTo-Json -Depth 6

# 6) 轮询状态（1未执行 2执行中 3异常结束 4正常结束）
for ($i=0; $i -lt 15; $i++) {
  $plan = Invoke-RestMethod -Method Get -Uri "http://localhost:8080/api/ExePlanController/getExePlanByName?planName=$planName" -Headers $headers
  Write-Host "state=$($plan.exeState), lastError=$($plan.lastError)"
  if ($plan.exeState -eq 3 -or $plan.exeState -eq 4) { break }
  Start-Sleep -Seconds 1
}

# 7) 查询执行结果
$result = Invoke-RestMethod -Method Get -Uri "http://localhost:8080/api/AlgRltSaveController/getAlgSaveByAlgName?planId=$planId&algId=$($alg.algId)&algName=$($alg.algName)-1" -Headers $headers
$result | ConvertTo-Json -Depth 8
```

## 4. 验收标准

- `execute` 返回 `code=200` 且 `data.accepted=true`
- 计划最终状态 `exeState=4`
- 结果查询返回非空数组，包含 `key/value/dataType` 字段

## 5. 失败排查

1. `accepted=false`：计划可能已在执行或队列已满。  
2. 计划状态变为 `3`：查看 `lastError`，重点检查算法服务名、Nacos 注册、`/myAlg/` 是否可用。  
3. 无结果：确认查询时 `algName` 需使用 `算法名-序号`（如 `demo-java-alg-001-1`）。
4. `addExePlan` 返回 500：优先检查是否计划重名（脚本已改为时间戳命名）；若仍失败，查看后端日志：`docker logs --tail 300 c_exphlp_webapp`。  
5. `execute` 提示 `planId不能为空`：说明上一步没有拿到有效 `planId`，不要继续执行第 5-7 步，先修复第 4 步。
