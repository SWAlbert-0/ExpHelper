# 快速上手：从部署到执行

本文档面向首次使用者，目标是在最少前置知识下跑通一条完整链路：

部署平台 -> 问题实例入库 -> 算法入库 -> 执行计划 -> 查看日志与结果。

## 1. 前置条件

- 已安装 Docker Desktop（或 Docker Engine）
- 可用端口：`8080`、`8086`、`8848`

## 2. 一键部署

在仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action deploy
```

首次机器建议：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action bootstrap
```

## 3. 运行检查

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action check
```

访问地址：
- 前端：`http://localhost:8086`
- 后端健康检查：`http://localhost:8080/api/auth/healthz`
- Nacos：`http://localhost:8848/nacos`

## 4. 问题实例入库（网页）

菜单：`问题实例管理`

两种方式：
1. 点击“添加”手工录入
2. 点击“JSON导入”拖拽 `.json` 文件导入
3. 点击“查看操作手册”可直接在弹窗内查看对应章节

建议至少填写：
- 实例名称（后续计划中用于选择）
- 分类（用于组织展示）

## 5. 算法入库（网页）

菜单：`算法库管理`

两种方式：
1. 点击“添加”手工录入
2. 点击“JSON导入”拖拽 `.json` 文件导入
3. 点击“查看操作手册”可直接在弹窗内查看接入说明

关键要求：
- `服务名` 必须与 Nacos 注册名一致
- `运行时` 选择与算法服务实际语言一致（`java/python`）

如需接入自己的算法项目，可直接基于模板复制：
- `docs/templates/java-springboot-nacos`
- `docs/templates/python-fastapi-nacos`

页面提示建议：
- 在“问题实例管理”完成入库后，按页面顶部“下一步”提示进入“算法库管理”。
- 在“算法库管理”完成入库后，按页面顶部“下一步”提示进入“执行计划管理”。

## 6. 执行计划

菜单：`执行计划管理 -> 执行向导`

推荐流程：
1. 选择问题实例
2. 选择算法并配置参数
3. 点击“执行检查”
4. 检查通过后提交并执行

补充：可点击“查看操作手册”弹窗查看本流程说明。

## 7. 查看执行过程与结果

- 在计划详情中查看“执行日志”（支持导出 CSV/JSON）
- 在“执行结果”弹窗查看指标汇总与明细
- 通知相关可在“通知记录”查看状态与失败原因，并执行补发

通知通道说明：
- 当前版本默认启用邮件通知链路；
- 微信通知通道为扩展预留，后续版本按需接入。

## 8. 用户常用命令

```powershell
# 一键重建并发布 webapp/front
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action deploy

# 启动算法服务容器
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-up -Mode offline

# 检查算法服务和Nacos实例
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-check
```

## 10. Python 案例快速验证（新增）

如需快速验证 Python 上传构建链路，可直接使用：

- `docs/cases/moo-python-nsga2-zdt1`
- `docs/cases/moo-python-moead-lsmop`

推荐顺序：
1. 导入 `problem_instances/*.json`
2. 导入 `algorithm_instances/*.json`（确认 `runtimeType=python`）
3. 执行 `scripts/package-source-zip.ps1` 生成源码 zip
4. 在算法库页面上传源码并构建启动
5. 在执行计划中通过“执行向导 -> 执行检查 -> 提交执行”
