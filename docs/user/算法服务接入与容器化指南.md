# 算法服务接入与容器化指南

本文档面向最终用户，说明如何把“你自己的算法服务”接入平台并在 Nacos 中稳定注册。

## 1. 目标与原则

- 目标：用户可上传/维护自己的算法服务，并在前端执行计划中直接调用。
- 原则：
  - 服务名必须全链路一致（代码注册名 = 算法库服务名 = 执行计划调用名）。
  - 优先使用容器后台常驻，不依赖命令行窗口存活。

## 2. 最小接入要求

你的算法服务需要满足：

1. 提供 HTTP 接口：`POST /myAlg/`
2. 入参结构兼容 `AlgRunCtx`（包含 `probInst`、`runParas`、`outPuter`）
3. 返回 `EachResult[]`（至少包含可解析的 Pareto 点或关键指标）
4. 能注册到 Nacos（Spring Cloud Alibaba Nacos Discovery）

建议技术栈：
- Java 17
- Spring Boot 2.7.18
- Spring Cloud 2021.0.8
- Spring Cloud Alibaba 2021.0.5.0

## 3. 上传你的算法服务（推荐流程）

将你的算法服务目录放到：

`docs/cases/<your-case>/algorithm_service/`

目录建议结构：

```text
algorithm_service/
  pom.xml
  Dockerfile
  src/main/java/...
  src/main/resources/application.yml
```

`application.yml` 至少包含：

- `spring.application.name: ${ALG_SERVICE_NAME:<your-service-name>}`
- `spring.cloud.nacos.discovery.server-addr: ${NACOS_SERVER_ADDR:localhost:8848}`
- `server.port: ${ALG_PORT:<your-port>}`

## 4. 容器化运行（推荐）

### 4.1 接入 compose

在 `docker/docker-compose.yml` 添加一个服务块（参考已有 `*_example` 服务）。

关键点：
- 使用统一算法运行镜像 `exphlp-alg-runner:local`（单镜像多实例）
- 环境变量至少包含：
  - `ALG_PORT`
  - `ALG_SERVICE_NAME`
  - `NACOS_SERVER_ADDR`
  - `NACOS_NAMESPACE`
  - `NACOS_GROUP`
  - `ALG_TYPE`（`nsga2/moead/smpso/spea2`）

### 4.2 启动

统一命令（已集成多案例）：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-up -Mode offline
```

说明：
- 脚本会先在本机执行 `mvn -q -DskipTests package`，再构建容器。
- 这样可规避 Docker Hub 拉取 `maven` 构建镜像失败的问题。

可选模式（适配不同环境）：

- `-Mode auto`（默认）：自动选择最稳妥路径（优先本机打包）。
- `-Mode online`：在线模式，可配合 `-SkipLocalPackage` 跳过本机打包。
- `-Mode offline`：离线优先，本机先打包并依赖本地已有镜像/依赖。

示例：

```powershell
# 默认推荐（本机先打包，再单镜像多实例启动）
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-up -Mode offline

# 在线模式（可选）
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-up -Mode online -SkipLocalPackage
```

检查状态：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-check
```

停止服务：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-down
```

## 5. 在前端入库并执行

1. 算法库管理 -> 新增（或 JSON 导入）  
2. 服务名填写为 Nacos 注册名（必须一致）  
3. 执行计划管理 -> 添加计划 -> 选择算法 -> 配置参数 -> 执行检查 -> 提交执行  

## 6. 常见错误与处理

### 6.1 Nacos 无可用实例

- 检查服务名是否一致
- 确认算法容器状态为 `Up`
- 确认 Nacos 可访问：`http://localhost:8848/nacos`

### 6.2 Docker Hub 拉取超时

典型报错：
- `failed to fetch anonymous token`
- `connectex: ... failed to respond`

处理：
- 使用 `scripts/ops.ps1 -Action alg-up -Mode offline`（本机先打包，减少远端镜像依赖）
- 配置网络代理/镜像加速后重试

### 6.3 Maven 参数异常（`.repo.local=E`）

典型报错：
- `No plugin found for prefix '.repo.local=E'`

原因：
- PowerShell 参数拆分导致 `-Dmaven.repo.local=...` 被错误解析。

处理：
- 使用 `scripts/ops.ps1 -Action alg-up -Mode offline`（内部调用新版 `up-alg-services.ps1`，已改为进程参数数组调用 Maven）。

### 6.4 参数改大后超时

- 执行日志若出现 `ALG_CALL_TIMEOUT`，说明服务可达但计算超时。
- 降低 `populationSize/maxGenerations/runNum` 或提高 `ALG_CALL_READ_TIMEOUT_MS`。

## 7. 新机器/服务器快速部署

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action bootstrap
```

该脚本会自动完成：
1. 启动基础容器（mongo/rabbitmq/nacos/webapp/front）
2. 启动算法容器
3. 执行运行与算法服务健康检查
