# 常见问题与排障

本文档用于快速定位高频问题。建议先执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action check
```

## 1. 执行前检查失败：Nacos 中无可用实例

现象：
- 执行向导提示服务无可用实例

检查顺序：
1. 算法库 `serviceName` 是否与 Nacos 服务名一致
2. Nacos 控制台是否有健康实例
3. 算法容器是否处于 `Up` 状态

修复命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-up -Mode offline
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action alg-check
```

## 2. 源码构建失败：wait nacos registration timeout

现象：
- 构建日志显示镜像和容器已启动，但最终失败

处理：
1. 重建 `webapp` 使最新构建脚本生效
2. 重新上传并构建

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ops.ps1 -Action deploy
```

## 3. 执行结果弹窗 404 或结果为空

常见原因：
1. 后端容器非最新镜像
2. 算法服务返回结构不完整
3. 计划未真正执行完成

处理：
1. 查看 `http://localhost:8080/api/auth/healthz` 的 `buildTime`
2. 查看执行日志是否有 `PLAN_DONE` / `PLAN_FAIL`
3. 必要时重建 `webapp`

## 4. 邮件通知未发送

检查执行日志 `MAIL_NOTIFY` 阶段的 `reasonCode`：
- `MAIL_CONFIG_INVALID`
- `MAIL_FROM_EMPTY`
- `MAIL_FROM_INVALID`
- `MAIL_SEND_FAILED`
- `MAIL_TENCENT_CONFIG_INVALID`
- `MAIL_TENCENT_SEND_FAILED`

修复后可在通知记录里补发。

## 5. 删除后页面列表不变

先点击“刷新”拉取最新数据；如果仍异常，检查接口返回中是否为：
- 已删除
- 记录不存在（幂等成功）
- 被执行计划引用（阻止删除）

## 6. 计划执行超时（Read timed out）

说明算法服务可达，但当前参数规模过大。  
优先降低：
- `runNum`
- `populationSize`
- `maxGenerations`

必要时再提升超时配置并重建后端容器。

## 7. Python 源码上传后构建失败

优先检查：
1. zip 根目录是否包含 `exphlp-alg.json`
2. `runtimeType` 是否与算法库一致（应为 `python`）
3. `serviceName` 是否与算法库、Nacos 注册名一致
4. 是否包含 `requirements.txt`（或 `pyproject.toml`）

推荐先用官方案例做对照验证：
- `docs/cases/moo-python-nsga2-zdt1`
- `docs/cases/moo-python-moead-lsmop`
