# 配置清单

## 1. 目的与范围
- 本文档是项目“运行时 + 部署”配置文件的唯一清单。
- 任何配置项的新增或修改，都必须同步更新本文档。
- 不在范围内：`node_modules/`、`target/`、`dist/`、普通依赖元数据文件。

## 2. 维护规则（强制）
- 新增配置文件时：先补充本清单，再提交代码。
- 修改现有配置用途时：同步更新“用途说明”。
- 提交前执行校验命令：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/check-config-doc.ps1
```

## 5. 构建依赖规则（强制）
- `docker/webapp/Dockerfile` 依赖本地产物：`exphlp/api/webApp/target/webApp-1.0-SNAPSHOT.jar`。
- 因此每次需要发布后端变更时，必须先重新打包 JAR，再执行镜像构建。
- 推荐命令（已内置在脚本中）：
```powershell
cd exphlp
mvn -q -DskipTests package
```
