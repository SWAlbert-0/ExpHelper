# 错误码映射（版本2）

> 文档状态：历史归档（版本2）  
> 用途：问题追溯与审计参考  
> 当前执行基准：`docs/dev/维护手册.md` 与 `docs/dev/配置清单.md`

## 统一响应结构

后端接口统一返回以下字段：

- `code`: 数值状态码（200/400/401/404/500）
- `msg` / `message`: 错误或成功描述
- `data`: 业务数据
- `errorCode`: 业务错误码（失败时）
- `traceId`: 请求链路标识
- `path`: 请求路径
- `timestamp`: 毫秒时间戳

## 关键业务错误码

| errorCode | code | 含义 |
| --- | --- | --- |
| `AUTH_MISSING_TOKEN` | 401 | 未携带 token |
| `AUTH_INVALID_TOKEN` | 401 | token 非法或过期 |
| `AUTH_UNAUTHORIZED` | 401 | 当前请求未认证 |
| `AUTH_INVALID_CREDENTIALS` | 401 | 登录用户名或密码错误 |
| `USER_EMAIL_INVALID` | 400 | 邮箱格式错误 |
| `USER_MOBILE_INVALID` | 400 | 手机号格式错误 |
| `USER_QQ_INVALID` | 400 | QQ 格式错误 |
| `PLAN_ID_EMPTY` | 400 | 执行计划 ID 为空 |
| `PLAN_ID_FORMAT_INVALID` | 400 | 执行计划 ID 格式错误 |
| `PLAN_NOT_FOUND` | 404 | 执行计划不存在 |
| `AVATAR_FILE_EMPTY` | 400 | 上传头像为空 |
| `AVATAR_FILE_TYPE_INVALID` | 400 | 头像格式不支持 |
| `AVATAR_UPLOAD_FAILED` | 500 | 头像上传失败 |
| `INVALID_ARGUMENT` | 400 | 全局参数非法异常 |
| `INTERNAL_ERROR` | 500 | 全局未捕获异常 |

说明：后端 `ApiResponse.failed(...)` 已约束 `errorCode` 必填，禁止空错误码返回。

## 执行日志阶段码（通知相关）

邮件通知结果不走独立 API 错误码，而通过执行计划日志阶段 `MAIL_NOTIFY` 反馈：

| stage | reasonCode | 含义 |
| --- | --- | --- |
| `MAIL_NOTIFY` | `MAIL_SENT` | 邮件发送成功 |
| `MAIL_NOTIFY` | `MAIL_NOT_CONFIGURED` | SMTP 未配置，邮件未发送 |
| `MAIL_NOTIFY` | `MAIL_SEND_FAILED` | SMTP 调用失败 |
| `MAIL_NOTIFY` | `MAIL_USER_NOT_FOUND` | 通知用户不存在 |
| `MAIL_NOTIFY` | `MAIL_NO_RECEIVERS` | 计划未配置通知用户 |
| `MAIL_NOTIFY` | `MAIL_OUTBOX_CREATED` | 异步通知任务已入队 |
| `MAIL_NOTIFY` | `MAIL_QUIET_HOURS_DELAYED` | 命中静默时段，通知延迟发送 |
| `MAIL_NOTIFY` | `MAIL_MANUAL_RESEND_CREATED` | 手动补发任务已创建 |
| `MAIL_NOTIFY` | `MAIL_USER_DISABLED` | 用户已关闭计划结束通知 |
| `MAIL_NOTIFY` | `MAIL_CONFIG_INVALID` | SMTP配置缺失（host/port） |
| `MAIL_NOTIFY` | `MAIL_FROM_EMPTY` | 发件人邮箱为空（from/username均未配置） |
| `MAIL_NOTIFY` | `MAIL_FROM_INVALID` | 发件人邮箱格式不合法 |
| `MAIL_NOTIFY` | `MAIL_TENCENT_CONFIG_INVALID` | 腾讯SES配置不完整或未启用 |
| `MAIL_NOTIFY` | `MAIL_TENCENT_SEND_FAILED` | 腾讯SES API调用失败 |
| `MAIL_NOTIFY` | `MAIL_PROVIDER_FALLBACK_USED` | SMTP失败后已切腾讯SES发送成功 |
| `MAIL_NOTIFY` | `MAIL_PROVIDER_ALL_FAILED` | SMTP与腾讯SES均发送失败 |

## 前端提示映射

前端在 `exphlp-front/src/utils/errorCode.js` 中按如下优先级映射提示文案：

1. `errorCode` 映射
2. `code` 映射
3. 后端 `msg/message`
4. 默认文案

前端额外约束：`/admin/*` 调用在请求层直接拦截，映射 `LEGACY_ADMIN_DISABLED`。
