## 迭代 01 - 部署链路重构与容器化收敛
### 目标与验收
- 目标：形成可稳定复现的本地部署路径，解决“镜像来源不一致、启动失败、前端构建受网络代理影响”问题。
- 验收：`webapp` 与 `exphlp_front` 能正常启动，`/api/auth/healthz` 与 `/api/auth/captcha` 可访问。

### 关键改动
- 将部署相关文件统一收敛到 `docker/`，文档统一收敛到 `docs/`。
- 重写 `docker/docker-compose.yml` 与 `docker/docker-compose.local-front.yml`，支持“全容器构建”和“本地dist兜底”。
- 新增 `docker/scripts/preflight.ps1` 进行镜像、端口、jar产物检查。
- 新增 `docs/新版部署方法.md`，固化启动步骤与故障速查。

### 重难点与报错闭环
- 报错：`base name (${FRONT_NGINX_IMAGE}) should not be blank`
  - 根因：环境变量缺失。
  - 修复：补齐 `.env` 必填键，并在预检脚本强校验。
- 报错：`npm ci ECONNREFUSED 127.0.0.1:7897`
  - 根因：容器内继承了不可用代理。
  - 修复：提供本地构建前端+挂载dist的兜底方案。
- 结果：前端与后端容器可稳定启动，部署流程可复现。

## 迭代 03 - 登录鉴权链路修复与验证码下线
### 目标与验收
- 目标：解决“登录后立刻401”“登录状态已过期循环弹窗”“undefined.data”报错；按需求下线验证码。
- 验收：`/api/auth/login -> /api/auth/me` 连续返回 200，登录页无验证码。

### 关键改动
- 前端 `request.js` 在 401 分支显式 `reject`，避免连锁JS异常。
- 登录流程移除验证码字段与接口依赖（`code/uuid/getCodeImg`）。
- 后端 `currentUser` 增加兼容兜底：按 `userId` 查不到时按 `userName` 查。
- 用户主键注解统一为 `@Id`，降低 Mongo 映射差异风险。

### 重难点与报错闭环
- 报错：`TypeError: Cannot read properties of undefined (reading 'data')`
  - 根因：401 分支只弹窗不 reject。
  - 修复：401 分支返回 `Promise.reject` 并跳转 `/login`。
- 报错：`/api/auth/me -> 401 未登录（login已200）`
  - 根因：用户ID查询与历史数据 `_id` 类型不一致。
  - 修复：DAO 双ID兼容 + `currentUser` 按用户名兜底。

## 迭代 06 - 版本2稳定性收口与接口规范统一
### 目标与验收
- 目标：在 `版本2` 分支统一错误响应结构、补齐关键链路日志字段、完善平台管理字段校验与最小 smoke 回归。

### 关键改动
- 后端：新增统一响应工具（`ApiResponse`）和请求链路标识工具（`TraceContext`），在 `AuthInterceptor/AuthController/ExePlanMgrCtrl/PlatMgrCtrl` 接入。
- 后端：为登录、资料更新、头像上传、计划执行、用户管理写入结构化日志字段（traceId/userId/path/errorCode）。
- 前端：`request.js` 支持 `errorCode` 优先映射与 `message` 兼容，401 弹窗增加并发防抖。
- 前端：平台管理页面新增邮箱/手机号/QQ 规则校验，提交前统一校验。
- 运维：`docker/scripts/smoke.ps1` 增加 `/api/auth/me`、头像上传校验和 `bad-id` 执行计划错误码断言。
- 文档：新增 `docs/dev/错误码映射-v2.md` 与 `docs/dev/接口总览-v2.md`。
- 残留下线（第一批）：移除顶部“消息通知”入口与 `/user/msg` 路由，避免误入依赖 `/admin/*` 的历史消息模块。
- 残留下线（第二批）：禁用动态模板路由注入（固定业务路由模式），并移除 `/dict` 隐藏路由入口，确保 `/admin/*` 历史模块不再通过主导航进入。

### 重难点与报错闭环
- 报错：前后端错误字段不统一（仅 `msg`），导致提示文案不稳定。
  - 根因：接口响应结构未统一，前端仅按 `code/msg` 处理。
  - 修复：后端统一输出 `errorCode + traceId + message`，前端按 `errorCode` 优先映射。
  - 验证：登录失败、无 token、bad-id 执行计划可返回稳定文案与 traceId。
- 报错：平台管理前后端校验规则不一致。
  - 根因：页面缺少完整规则，后端接口无字段校验。
  - 修复：前后端统一邮箱/手机号/QQ/密码长度规则。
  - 验证：非法字段提交被拦截，合法字段可保存。

## 迭代 10 - 门禁可靠性与Playwright最小闭环
### 目标与验收
- 目标：修复 `check-p0` 假通过风险，补齐 smoke 的 PowerShell 5.1 兼容性，并接入 Playwright 最小链路测试。
- 验收：任一步骤失败可被门禁真实阻断；smoke 的头像上传步骤通过；Playwright 可执行并包含至少 1 条登录主链路用例。

### 关键改动
- 门禁：`scripts/check-p0.ps1` 改为 fail-fast，原生命令非0退出码会触发失败。
- 门禁：`.github/workflows/p0-check.yml` 同步更新 eslint 目标路径（`views/profile/userInfo.vue` + `src/api/auth.js`）。
- smoke：`docker/scripts/smoke.ps1` 将头像上传改为手工 multipart，兼容 PowerShell 5.1。
- e2e：前端新增 Playwright 配置与首条登录链路测试（Mock `login/me`）。
- CI：新增 `.github/workflows/playwright-e2e.yml`。

### 重难点与报错闭环
- 报错：`check-p0` 历史上可能出现“失败仍 PASS”。

    - 增加注册/心跳无限重试（异常不退出线程）；
    - `requests.Session().trust_env = false`，忽略容器代理环境；
    - `docker run` 时显式清空 `HTTP_PROXY/HTTPS_PROXY` 并补 `NO_PROXY`。
- 后端运行时运维：

  - `PRUNE_IMAGES` 扩展为“清理旧资源”：先删同算法下已停止容器，再按保留策略清理旧镜像，并返回 `removedContainers/removedImages/skippedRunningContainers`。
