# NSGA2-ZDT1 网页操作步骤（重写版）

本手册只讲“怎么点页面”。每一步都给出可复制的示例值和完成判定。

## 0. 名称约定（先看）

- 算法服务注册名（Nacos）：`nsga2-zdt1-ls`
- 网页算法名称：建议也使用 `nsga2-zdt1-ls`
- 网页服务名：必须填写 `nsga2-zdt1-ls`

如果服务名写错，即使算法服务已启动，也会报：
- `Service Instance cannot be null`
- 或 `服务[...]在Nacos中无可用实例`

## 1. 启动依赖服务

1. 确保平台前后端已启动（`8086`/`8080`）。
2. 启动 Nacos，确认可访问 `http://localhost:8848/nacos`，并确认 `9848` 端口对主机可达（Nacos 2.x 注册必需）。
3. 启动示例算法服务（推荐一键脚本）：

```powershell
powershell -ExecutionPolicy Bypass -File docs/examples/moo-nsga2-zdt1/scripts/start-alg-with-nacos.ps1
```

完成判定：
- Nacos 服务列表中可看到 `nsga2-zdt1-ls`。

如果你坚持手动启动，可使用：
```powershell
cd docs/examples/moo-nsga2-zdt1/algorithm_service
mvn spring-boot:run
```
但手动方式更容易出现端口占用、Nacos地址不一致等问题。

## 2. 新增问题实例

页面：`问题实例管理` -> `添加`

示例值：
- 实例名称：`zdt1-ls100-caseA-ui`
- 类别名称：`moo-zdt1`
- 目录名称：`examples/zdt1/ls100/caseA`
- 机器名称：`local`
- IP地址：`127.0.0.1`
- 实例描述：`ZDT1 large-scale baseline; nVars=100`

完成判定：
- 列表中出现 `zdt1-ls100-caseA-ui`。

## 3. 新增算法

页面：`算法库管理` -> `添加`

示例值：
- 算法名称：`nsga2-zdt1-ls`
- 服务名：`nsga2-zdt1-ls`
- 算法描述：`NSGA-II large-scale ZDT1 example`

参数列表逐条新增：
1. `nVars` / `int` / `100`
2. `populationSize` / `int` / `200`
3. `maxGenerations` / `int` / `250`
4. `crossoverProbability` / `double` / `0.9`
5. `mutationProbability` / `double` / `0.01`
6. `seed` / `long` / `20260223`

完成判定：
- 算法列表出现 `nsga2-zdt1-ls`
- 参数类型下拉可见 `long`
- 该算法的服务名显示为 `nsga2-zdt1-ls`

## 4. 创建执行计划

页面：`执行计划管理` -> `添加`

步骤 1（基本信息）：
- 计划名称：`plan-zdt1-ls100-ui`
- 勾选问题实例：`zdt1-ls100-caseA-ui`
- 计划描述：`NSGA-II + ZDT1 UI test`

步骤 2（算法配置）：
- 选择算法：`nsga2-zdt1-ls`
- 点击 `新增`
- 运行次数设为 `1`

步骤 3（通知人员）：
- 任选至少 1 位用户

点击 `提交`。

完成判定：
- 列表出现新计划，状态为 `未执行`。

## 5. 执行与查看

1. 在计划列表点击 `执行`。
2. 观察状态流转：
   - `未执行 -> 执行中 -> 正常结束/异常结束`
3. 点击 `查看`：
   - 在算法条目点击 `执行结果`，检查包含：
     - 状态与原因码（SUCCESS/PARTIAL/EMPTY/MISSING）
     - 指标汇总：`Runtime(ms)`、`HV`、`IGD+`、`Spread(Δ)`
     - 每个 run 的明细指标与 `Pareto点数`
4. 点击 `执行日志`：
   - 日志每 2 秒增量刷新
   - 默认只看“最新执行”批次，也可切换“全部历史”
   - 可查看阶段：`PLAN_START`、`ALG_CALL`、`RETRY`、`PLAN_DONE`、`PLAN_FAIL`
   - 支持导出 `CSV`/`JSON`（按钮：`导出CSV`、`导出JSON`）

## 6. 异常处理

如果状态为 `异常结束`：
1. 在列表“异常原因”列或详情中先看错误。
2. 在“执行日志”看失败阶段与具体提示。
3. 常见修复：
   - 算法服务未启动：启动算法服务并确认 Nacos 有 `nsga2-zdt1-ls`
   - 服务名不匹配：算法库中的服务名改回 `nsga2-zdt1-ls`
   - 向导执行检查报 `系统接口404异常`：说明后端版本未包含 `wizardPrecheck`，请重建并启动最新 `webapp` 后重试
4. 修复后点击 `重新执行`。

## 7. 删除与同步校验（新增）

用于验证“前后端一致性”：

1. 在 `问题实例管理` 选择一个未被任何计划引用的实例，点击 `删除`。
2. 预期：
   - 出现“删除成功”提示；
   - 当前表格该行立即消失（无需手动刷新）。
3. 再尝试删除一个被计划引用的实例。
4. 预期：
   - 提示“问题实例已被执行计划引用”；
   - 该行保留（属于业务保护，不是删除故障）。

说明：
- 若提示“记录已不存在，列表已同步”，表示该记录已在后端不存在，前端已做同步刷新。
- 若提示“删除未生效，请刷新后重试”，表示后端未确认删除，需要排查后端日志与数据状态。
