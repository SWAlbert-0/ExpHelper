 实验助手项目开发与部署说明 Docker Desktop 安装 1. 开启 WSL 服务 以管理员身份运行 Windows PowerShell ，输入 wsl ，检查是否本机开启 wsl 服务 ： 若未开启，则需按照以下步骤进行配置：右键单击 Windows 按钮并选择 “ 应用和功能 ” → 选择相关设置下右侧的 “ 程序和功能 ” → 选择 “ 打开或关闭 Windows 功能 ” → 选择 “Hyper-V” ，然后单击 “ 确定 ” （注意：安装完成后，系统会提示你重新启动计算机 ， 此步骤针对 win 10 系统 ） 2. 下载 Docker Desktop 官方下载地址： HYPERLINK "https://www.docker.com/products/docker-desktop/" https://www.docker.com/products/docker-desktop/ 安装参考教程： HYPERLINK "https://www.quanxiaoha.com/docker/windows-install-docker.html" https://www.quanxiaoha.com/docker/windows-install-docker.html 安装完成，打开 Windows PowerShell ，分布输入 docker -- version 和 docker-compose – version ，出现版本号即安装成功 3. 更换 Docker Desktop 镜像源 打开 Docker Desktop ， 在 “ 设置 ” → “Docker Engine” 中，添加国内的镜像源 ： " registry -mirrors": [ "https://l346q0b2.mirror.aliyuncs.com" , "https://docker.mirrors.ustc.edu.cn" , "http://hub-mirror.c.163.com" ] 添加完成后，点击 “Apply & restart” ， 即可更换完成 项目部署 1. 可执行 jar 包构建 docker 镜像 （ 1 ） 新建文件夹 “ project _to_docker ” （ 2 ） 在 “ project _to_docker ” 文件夹下新建 Dockerfile 文件 ， 并 输入以下内容 ： # 使用官方 OpenJDK 8 映像作为基础镜像 FROM openjdk:8-jdk-alpine # 复制应用程序的 JAR 文件到容器中 ADD webApp-1.0-SNAPSHOT.jar webApp-1.0.jar # 暴露应用程序的端口 EXPOSE 8080 # 启动 Spring Boot 应用程序 CMD [ "java" , "-jar" , "webApp-1.0.jar" ] （ 3 ） 将 webApp-1.0-SNAPSHOT . jar 文件放入 “ project_to_docker ” 文件夹下 （ 4 ） 在 “ project_to_docker ” 文件夹下打开 CMD 命令行 ， 执行命令（其中 webapp 为 docker 镜像名，必须小写； “ . ” 表示 Dockerfile 在当前目录下） ： docker build -t webapp . （ 5 ） 执 行完毕后，可在 Docker Desktop 中看到刚刚创建的 名为 “ webapp ” 的 镜像 2. 拉取 MongoDB 镜像 （ 1 ） 打开 CMD 命令行，执行命令（从官方库拉取 mongo 镜像， latest 表示最新版本）： docker pull mongo:latest （ 2 ） 执行完毕后，可在 Docker Desktop 中看到刚刚拉取的 mongo 镜像 3. 拉取 RabbitMQ 镜像 （ 1 ） 打开 CMD 命令行，执行命令（从官方库拉取 rabbitmq 镜像， management 表示带有管理页面）： docker pull rabbitmq:management （ 2 ） 执行完毕后，可在 Docker Desktop 中看到刚刚拉取的 rabbitmq 镜像 4. 拉取 Nginx 镜像 （ 1 ） 打开 CMD 命令行，执行命令（从官方库拉取 nginx 镜像， latest 表示最新版本）： docker pull nginx:latest （ 2 ） 执行完毕后，可在 Docker Desktop 中看到刚刚拉取的 nginx 镜像 5 . 拉取 N acos 镜像 （ 1 ） 打开 CMD 命令行，执行命令（从官方库拉取 nginx 镜像， latest 表示最新版本）： docker pull n acos / nacos -server （ 2 ） 执行完毕后，可在 Docker Desktop 中看到刚刚拉取的 nginx 镜像 6 . 构建前端的 docker 镜像的准备步骤 （ 1 ） 将 nginx-docker 压缩包解压到 D 盘 （ 2 ） 将 dist 文件夹放入 nginx-docker 文件夹下的 html 文件夹中（若已有则进行替换） 7 . 用 docker - compose 脚本进行容器编排 （ 1 ） 新建文件夹，命名为： exphlp （ 2 ） 新建 docker - compose .yml 文件，并写入以下内容： version : '3' services : #
